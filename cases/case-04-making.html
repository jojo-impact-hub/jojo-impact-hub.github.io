辦公室許願牆

Codex Firebase Firestore 即時同步 四步上線

用 Codex（OpenAI VS Code 延伸模組）生成程式碼，串接 Firebase Firestore 實作即時留言牆，全程不寫一行程式碼，從零到上線約 30 分鐘。

為什麼需要 建置流程 指令範本 資料結構 踩雷紀錄 後續更新

1 為什麼需要這個工具

痛點 1

想收集團隊心聲，用 Google Forms 收不到即時感——填完看不到其他人說什麼

痛點 2

靜態網頁存 localStorage，只存在自己的瀏覽器，換裝置或換人就看不到

痛點 3

需要「雙向」——不只是送出資料，還要即時看到所有人的留言

痛點 4

不想管伺服器，希望免費、穩定、部署簡單

2 建置流程

1
用 Codex 生成 index.html
在 VS Code 右側 Codex 面板輸入需求，1 分 16 秒產出單一 HTML 檔案，包含暱稱欄、留言欄、送出按鈕、Firestore 串接框架，並在程式裡標注「把 Firebase 設定貼在這裡」的位置。
2
建立 Firebase 專案與 Firestore
到 console.firebase.google.com → 建立專案「office-wish」→ 開 Firestore Database（Standard 版，asia-east1，測試模式）。
3
拿 Firebase 設定，交給 AI 貼入
在 Firebase Console 左上角齒輪 → 一般 → 你的應用程式 → 點網頁（</>）→ 複製 firebaseConfig。把設定整段貼給 Codex：「這是我的 Firebase 設定，幫我貼到 index.html 正確位置並接上資料庫」，AI 自動完成。
4
登入 Firebase CLI，部署上線
在 VS Code 終端機自己跑登入指令（這步 AI 做不到，因為要跳出瀏覽器讓你親自點「允許」）。登入後 init hosting，再 deploy，完成。

3 指令範本

指令 1 階段 1 · 生成程式碼
請用一個 index.html 檔，做一個「辦公室許願牆」網頁：畫面上有暱稱欄、留言欄、一個送出按鈕；送出後把留言存到 Firebase Firestore，並即時顯示所有人的留言（最新的排最上面）。請用最簡單、最少檔案的寫法，並在程式裡用註解清楚標出「把 Firebase 設定貼在這裡」的位置。

關鍵技巧：「最簡單、最少檔案」避免 AI 過度工程化；「標出貼入位置」讓非工程師知道下一步要做什麼，不用自己看懂程式碼。

指令 2 階段 3 · 串接 Firebase
這是我的 Firebase 設定，幫我貼到 index.html 正確位置並接上資料庫：

[貼上你的 firebaseConfig]

為什麼不自己貼：Firebase 設定有特定位置和引入方式，讓 AI 貼比自己找位置快，也不會貼錯。

指令 3 階段 4 · 部署（自己跑）
npx firebase-tools login
npx firebase-tools init hosting
npx firebase-tools deploy --only hosting

注意：login 這行要自己在 VS Code 終端機打，輸入 y 同意安裝，瀏覽器跳出來點允許。init 時 Public directory 輸入 .（點），Single-page app 選 N，Overwrite index.html 選 N。

4 Firestore 資料結構

欄位 型別 說明
nickname string 留言者暱稱
message string 留言內容
createdAt timestamp 留言時間，用於 orderBy 排序（最新在上）

Collection 名稱：wishes｜即時監聽用 onSnapshot，任何人送出留言，所有開著的頁面會立刻更新，不需要手動重新整理。

5 踩雷紀錄

雷 1 npx 問 Ok to proceed? 按錯選 n

首次跑 npx firebase-tools 會問是否安裝，預設是 n，直接按 Enter 就取消了。要輸入 y 再按 Enter 才會繼續安裝。

雷 2 deploy 報錯「Not in a Firebase app directory」

直接跑 deploy 前要先跑 init hosting 產生 firebase.json，少這步就會報錯。init 時 Public directory 填 .（當前資料夾），不要填 public。

雷 3 login 叫 AI 跑，回報 non-interactive

Firebase 登入需要跳出瀏覽器讓你親自點「允許」，AI 的終端機是非互動模式，做不到這步。login 那行一定要自己在 VS Code 終端機手動跑。

6 後續更新網站的方法

1
在 VS Code 修改 index.html（告訴 Codex 或 Claude Code 要改什麼，讓 AI 改）
2
在 VS Code 終端機跑：npx firebase-tools deploy --only hosting
3
等約 30 秒，https://office-wish.web.app 自動更新，不需要重新 init 或 login
